Concept

Skills

Skill 是可复用的工作流说明。它告诉 Codex 什么时候触发、先读什么、输出什么、怎么验证。

本页它是什么什么时候用可复制示例

What

它是什么

Skills 适合沉淀重复流程，例如工程 grilling、学术 pipeline、部署检查、代码 review。公开指南里要区分官方能力和本机自写 skills。

触发条件什么时候必须使用，什么时候不该使用。
输入合同需要仓库、目标、范围、预算、人工 checkpoint。
执行步骤先读哪些文件，是否允许 subagent，如何验证。
输出合同产物、证据、风险、下一步和本地限制。

When

什么时候用 / 不用

当任务模式稳定、输入输出明确、需要多次复用时用 skill。一次性的想法、未定边界和临时偏好，不应该急着做成 skill；先用普通任务跑通，再把稳定流程沉淀。

Examples

可复制示例

[$grill-with-docs]
仓库：/path/to/repo
目标：把功能 X 做成可发布版本。
请先读 AGENTS.md、README、相关源码，再追问关键边界。

Anatomy

Skill 应该沉淀什么

Skill 不是更长的 prompt，而是可复用的操作规程。官方 skill 机制通常围绕 SKILL.md 展开，并可配合 scripts/、references/、assets/。公开指南里要让读者知道：什么时候直接写 prompt，什么时候值得做 skill。

触发条件

明确什么时候必须使用，例如“用户提到 grill-with-docs 时先读 skill”。

上下文读取

规定先读哪些文档、是否读完整、引用的相对路径如何解析。

执行流程

把步骤写成可复用顺序：preflight、提问、实现、验证、收口。

输出合同

规定产物格式：报告、表格、ADR、代码变更、测试证据。

Lifecycle

从 prompt 到 skill 的生命周期

先手写 prompt

第一次做任务不要急着写 skill。先用普通任务验证流程是否合理。

提取稳定步骤

连续两三次重复使用的内容才值得沉淀：输入、边界、检查、输出。

加入验证

skill 里必须写失败时怎么办、如何验证产物、什么时候需要人工确认。

维护版本

当项目规则变化时更新 skill；公开发布时标注哪些是 local adaptation。

Template

一个实用 Skill 模板

---
name: my-review-flow
description: Review a repo change for behavior regressions, test gaps, and docs drift.
---

Use when:
- The user asks for review, PR readiness, or risk audit.

Before acting:
- Read AGENTS.md and nearest project instructions.
- Inspect git diff and relevant tests.
- Do not edit files unless explicitly asked.

Process:
1. Identify changed behavior.
2. Check tests and docs.
3. Rank findings by severity.
4. Report only actionable issues with file evidence.

Output:
- Findings first.
- Then open questions.
- Then verification gaps.
