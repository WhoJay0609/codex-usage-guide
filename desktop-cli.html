我该用 Desktop 还是 CLI

Desktop 更适合交互式工程协作，CLI 更适合终端、自动化和权限可控的批处理。

Desktop

  • 适合多线程、浏览器检查、review pane、GitHub Pages、视觉反馈。
  • 第一条任务先读项目上下文，再做小范围修改和验证。
  • 不要用来跑无人值守批处理或把同一任务塞进多个重复线程。

CLI

  • 适合终端执行、codex exec、MCP、subagents、CI 辅助。
  • 第一条任务codex "read AGENTS.md and explain how to run tests"
  • 不要用来处理需要频繁视觉判断、截图比对、页面排版微调的任务。

选择规则

我要看页面Desktop。
我要脚本化CLI 或 codex exec。
我要改当前编辑器文件IDE。
我要处理 PR/issueGitHub integration。

真实实例：这次应该用哪个入口

用户在 Desktop 浏览器里指出网页内容问题，Codex 需要读取本地仓库、编辑 HTML、运行静态服务、用 curl 抽样验证，再推送 GitHub Pages。因此最合适的入口是 Desktop 主线程配合本地终端；如果只是批量检查所有链接，才适合沉淀成 CLI 或 codex exec 自动化。

入口选择：
Desktop: 接收网页反馈、解释修改、看发布 URL
Terminal: rg / git diff --check / http.server / curl
GitHub: push 后确认 Pages built
