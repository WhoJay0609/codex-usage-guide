Concept

Goal

Goal 是长任务的执行合同，不是标题。它把目标、范围、验收、验证和停止条件写清楚。

本页它是什么什么时候用可复制示例

What

它是什么

Goal 适合跨阶段、跨仓库、需要 artifact、需要 subagent 或需要 closeout 的任务。好的 Goal 会保留成功标准、验收标准、验证命令和停止条件。

When

什么时候用 / 不用

任务超过一个短回合、需要跟踪状态或需要最终验收时用 Goal。简单问答、单文件小改、纯 review 通常不需要。

Examples

可复制示例

Objective:
把指南拆成多页面 GitHub Pages 文档站。

Scope:
- Include: HTML pages, assets/site.css, assets/site.js
- Exclude: PDF 和 LaTeX

Acceptance:
- 每个页面可访问，导航可跳转
- GitHub Pages built

Stop Conditions:
- 官方资料无法访问
- 共享资源导致页面不可读
