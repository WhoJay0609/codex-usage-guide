Concept

Codex 是什么

Codex 是面向软件工程任务的编码代理体系。你可以在 Desktop、CLI、IDE、Cloud 和 GitHub 集成里使用它，但每个入口适合的任务不同。

本页它是什么什么时候用可复制示例

What

它是什么

Codex 不是单一聊天窗口，而是一组工作面：Desktop 适合可视化、多线程和本地仓库协作；CLI 适合终端、脚本和自动化；IDE 适合编辑器内上下文；Cloud/GitHub 适合 PR、issue 和远程任务。

Desktop本地仓库、网页预览、截图、review pane、长对话。
CLI终端执行、codex exec、MCP、CI 前置检查。
IDE编辑器里的局部代码修改和上下文引用。
GitHubissue、PR、review comment 和远程协作。

When

什么时候用 / 不用

  • 用 Desktop需要看 diff、页面、截图、浏览器和多线程进度。
  • 用 CLI需要脚本化、受控权限、MCP 或 codex exec。
  • 不用 Codex任务没有明确仓库、没有验收标准，或你不愿让代理读取相关上下文。
