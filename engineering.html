Scenario handbook

工程 flow

工程场景的目标不是“让 Codex 写更多代码”，而是把一个模糊 idea 变成可验证、可 review、可合并的最小变更。先 grill，再定边界，再拆 issue，再实现，再 review。

Lifecycle

从 idea 到可合并变更

复杂工程任务不要直接进入实现。先把需求、边界、失败条件和验收证据问清楚。

Idea intake

用户提出模糊目标，例如“改一下权限系统”。Codex 先读文档和代码，判断这是讨论、规划还是执行任务。

Grill 边界

使用 grill-with-docs 读取 AGENTS.md、README、CONTEXT、ADR 和相关源码，追问会影响结构的关键问题。

PRD / Issues

把稳定结论拆成小任务。每个 issue 都应该有范围、验收、测试和不做什么。

Implement

一次只做一个 issue。先改最小 diff，再跑最窄测试，不把 refactor 和 feature 混在一起。

Review

让 Codex 站在 reviewer 角度查行为回归、测试缺口、权限风险、文档漂移。

Ship

提交、推送、PR 或发布后，用 check 状态、页面访问、日志或用户验收完成 closeout。

Skills flow

工程 skills 怎么串起来

不要把 skill 当魔法词。每个 skill 都应服务于工程阶段，而不是替代判断。

ask-matt

用于路由：当前是 idea、PRD、issue、implement、review，还是需要回到 grilling。

grill-with-docs

用于边界澄清：读项目文档和代码，追问会影响架构、接口、验收的关键问题。

grill-me

用于强审设计：质疑方案是否过度、是否破坏边界、是否有更窄实现。

PRD / issue

把已稳定的需求变成可执行工单。不要让实现任务继续承载开放问题。

Implement

按 issue 做最小修改。每个实现回合都应能被测试或人工检查。

Review loop

用只读 review 查风险，再修复高优先级问题。不要把 review 变成无边界重构。

Prompts

可复制工程提示词

下面这些模板覆盖从澄清到实现的主要阶段。

边界澄清

[$grill-with-docs]
仓库：/path/to/repo
目标：把功能 X 做成可发布版本。

请先读：
- AGENTS.md
- README.md
- CONTEXT.md
- 相关源码和测试

请追问会影响结构的关键问题：
- 用户是谁？
- 公开接口是否能变？
- 兼容性要求是什么？
- 验收证据是什么？

不要开始实现，直到边界稳定。

实现一个 issue

请实现 issue #12。

范围：
- 可以改 src/checkout 和 tests/checkout
- 不要改 payment provider 接口

要求：
- 先读 AGENTS.md 和 issue 描述
- 给 3 行短计划
- 实现最小 diff
- 运行 npm test -- checkout

完成报告：
- 改了什么
- 验证结果
- 风险和未完成项

Subagents

工程任务什么时候拆给 subagent

工程任务适合把“调查”和“验证”拆出去，但核心产品判断、架构取舍和最终合并应由主线程负责。

适合拆

  • 只读审查安全、测试缺口、文档漂移、性能风险。
  • 并行验证不同浏览器、不同测试包、不同配置。
  • 独立切片互不重叠文件，例如 docs 和 CSS 分开。

不要拆

  • 同一核心文件多个代理同时改状态管理、schema、API contract。
  • 架构判断模块边界、公开接口、迁移策略应由主线程整合。
  • 外部写入发布、推送、评论、生产配置不交给 subagent 独立执行。

Real example

真实实例：把这份指南扩成实战手册

这次任务就是一个小型工程 flow：用户先指出内容太少，然后要求每个内容都有真实实例。Codex 没有直接“随便加文字”，而是读取项目规则、确认目标、分批修改页面、运行静态检查、提交推送并验证 Pages。

输入

坏输入是“帮我扩一下”；修复后写成目标、范围、读者、禁区和验收。

先读

项目规则、目标页面、现有导航、验证脚本和发布说明。

应动文件

HTML 内容、必要 CSS、验证脚本；不顺手改 CI/CD 或无关仓库。

验证

make check、空白检查、关键页面抽样和 diff review。

最终回答

按文件汇总改动、验证结果、未跑项和可合并风险。

失败停止

需求漂移、测试不可运行、写域变宽或出现无关 dirty 冲突时停止。
