Concept

权限与安全

权限决定 Codex 能读什么、写什么、运行什么命令、能不能联网。授权前先确认必要性和回滚方式。

本页它是什么什么时候用可复制示例

What

它是什么

sandbox、approval、network、auth 和 secret 管理共同构成安全边界。默认应该用最小权限：能只读就不写，能限定工作区就不开放 full access。

When

什么时候批准

  • 批准安装依赖、访问 GitHub API、启动本地服务、验证线上页面。
  • 拒绝理由不清、路径过宽、可能触碰 secret、命令具有破坏性。
  • 要求说明让 Codex 说明为什么需要权限、会改哪里、如何验证。

Examples

可复制示例

在运行任何需要网络或写权限的命令前，
请先说明：
1. 为什么需要这个权限；
2. 会读写哪些路径；
3. 失败后如何恢复；
4. 成功后如何验证。
