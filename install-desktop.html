    安装

    安装 Codex Desktop

    安装页的目标不是替代官方文档，而是告诉你第一次打开桌面版以后怎么把账号、仓库、权限和第一个任务串起来。下载入口和平台要求以 OpenAI 官方 Codex App 页面为准。

    ↗（外部链接）打开官方 Codex App 页面 查看桌面版操作

    安装前

    安装前先确认三件事#

    不要从第三方下载器或不明链接安装。先确认你用的是官方入口、合适的账号，以及准备让 Codex 读取的本地仓库。

    官方入口#

    从 ↗（外部链接）developers.openai.com/codex/app 进入，按官方页面当前展示的下载或打开方式安装。

    账号和组织#

    使用你实际工作的 OpenAI/ChatGPT 账号。团队项目要确认当前 workspace 或组织权限是否能访问 Codex。

    仓库准备#

    本地仓库最好已有 README、测试命令和 AGENTS.md。没有也能用，但 Codex 更容易猜错边界。

    安装流程

    安装到第一个任务#

    下面是建议的首次使用顺序。你可以把它当作安装后的检查单。

    第一个任务

    第一个任务怎么写#

    第一个任务应该是只读或小改动任务，用来确认 Codex 能读仓库、能理解规则、能运行验证，而不是一上来让它重构整个项目。

    只读入门#

    我刚安装 Codex Desktop，正在打开这个仓库：
/path/to/repo

请先只读分析：
- 读取 AGENTS.md 和 README
- 找到测试命令和主要入口
- 解释这个仓库适合怎样让 Codex 协作

不要改文件。

    小改动入门#

    请做一个小范围文档修改：
- 先读 AGENTS.md
- 只改 README 的安装说明
- 改动前告诉我计划
- 完成后运行最窄检查
- 最终说明 diff 和未验证项

    第一次不要这样写#

    不要写“帮我优化整个项目”“你自己看着办”“把所有问题都修了”。这些请求范围太大，无法判断 Codex 是否真的理解了仓库和你的验收标准。

    审批

    看到权限请求时怎么判断#

    Desktop 的一个优势是你能看到 Codex 为什么要申请某个动作。批准前重点看三点：动作是否必要、会影响哪里、完成后怎么验证。

    本地服务#

    比如启动 python3 -m http.server 检查静态页面。通常合理，但要确认只绑定本机地址。

    联网查询#

    比如读取官方文档或 GitHub Pages 状态。要求 Codex 说明来源和是否只读。

    安装依赖#

    要确认依赖来源、版本和是否会改 lockfile。文档任务通常不应随便安装新依赖。

    Git push#

    这是外部写操作。批准前应已经完成本地检查，并知道要推送哪个分支。
