Concept

MCP / Plugins

MCP 和 plugins 让 Codex 连接外部工具，例如 GitHub、设计工具、数据库、文档库和搜索服务。

本页它是什么什么时候用可复制示例

What

它是什么

MCP 是工具接口层，plugins 是打包后的能力组合。它们让 Codex 不只读本地文件，还能操作外部系统。

读外部状态GitHub PR、设计稿、文档库、数据库 schema。
写外部系统创建 issue、同步设计、部署配置，必须先确认。
权限来源token、账号、workspace、server 配置都要可追溯。
失败策略外部工具失败时停下报告，不伪造结果。

When

什么时候用 / 不用

需要实时外部状态、设计同步、GitHub 操作、数据库查询时使用。涉及 secret、生产数据或写操作时，必须先确认权限边界；如果本地文件已经足够完成任务，不要为了“更自动化”引入外部工具。
